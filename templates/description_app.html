Indices & Stocks Page

Overview

"Indices & Stocks" page is the core of the application, offering a real-time view of financial markets with data updated every 5 minutes.

Main Features

  • Indices Band : Display of the 8 major global indices (CAC 40, S&P 500, NASDAQ, Dow Jones, FTSE 100, Nikkei 225, DAX, Euro STOXX 50)
  • Interactive Chart : Visualization of stock prices with Chart.js, supporting 9 timeframes (1 day to 5 years)
  • Selected Stocks : 30 major stocks including US tech, CAC 40, and Asian stocks
  • Automatic Refresh : Automatic update every 5 minutes with countdown timer
  • API Status Indicator : Real-time monitoring of connectivity and response times

Used APIs

Yahoo Finance API

  • Main Endpoint : https://query1.finance.yahoo.com/v8/finance/chart/
  • Data Retrieved : Price, changes, volumes, historical data
  • Optimizations : Connection pool (20), parallel queries, timeout 5s
  • User-Agent : Browser simulation to avoid API blocking

Disadvantages and Limitations

🔴 Limitations of Yahoo Finance

  • Unofficial API : Yahoo Finance does not offer an official API, risk of blocking
  • Instability : Frequent changes in JSON structure, ongoing maintenance required
  • Rate Limits : No guarantee of availability, risk of rate limiting
  • Limited Data : No access to level 2 data, bid/ask spreads
  • Delays : Data with 15-20 minutes delay for US markets

🟡 Solutions for Improvement

  • Paid APIs : Bloomberg Terminal, Reuters Eikon, IEX Cloud Pro
  • Real-time Data : Finnhub Pro
  • Multiple Backup : Implementation of multiple data sources
  • Intelligent Caching : Caching data to reduce API calls
  • Advanced Monitoring : Alert system in case of API failure

Data Integrity Tests

Validation of API Responses

  • Verification of JSON structure of responses
  • Control of presence of required fields (price, change, change_percent)
  • Validation of timestamps and historical data

Error Handling

  • Timeout of 5 seconds per request
  • Automatic retry on failure
  • Display of explicit errors to user
  • Fallback to empty data in case of problem

Real-time Monitoring

  • Measurement of API response times
  • Visual status indicator (green/orange/red)
  • Counter of retrieved data (30 stocks)
  • Timestamp of last updates

Technical Architecture

Backend (Flask)

  • Main Route : /api/market-data for real-time data
  • Graphical Route : /api/chart-data-v2/<symbol> for historical data
  • Parallel Retrieval : ThreadPoolExecutor with 10 workers
  • Response Format : Structured JSON with separate indices and stocks

Frontend (JavaScript)

  • Automatic Refresh : Timer of 5 minutes with countdown
  • Interactive Chart : Chart.js with tooltips and zoom
  • State Management : Global variables for symbol and timeframe
  • Formatting : Formatting functions for prices and percentages

Call & Put Page

Overview

The "Call & Put" page is an advanced European option calculator using two pricing models: Black-Scholes (analytical) and Monte Carlo (numerical simulation) with interactive visualization of trajectories.

Main Features

  • Pricing of European Options : Call and Put with customizable parameters
  • Double Calculation Model : Black-Scholes (analytical) and Monte Carlo (simulation)
  • Greeks Calculation : Delta, Gamma, Theta, Vega, Rho for both models
  • Monte Carlo Simulation : Up to 10,000 trajectories with visualization
  • Confidence Interval : Statistical estimation of Monte Carlo precision
  • Interactive Visualization : Graph of trajectories and distribution of final prices
  • Progress Bar : Animation during calculations with performance metrics

Used APIs

Option Calculation API

  • Endpoint : /api/calculate-option (POST)
  • Parameters : Spot price, strike, maturity, risk-free rate, volatility, option type
  • Monte Carlo Parameters : Number of simulations, time steps, displayed paths
  • Response : Price, Greeks, confidence interval, trajectories, calculation time

Tradier API Integration

  • Real Option Data : Live option chains from Tradier for validation
  • Implied Volatility Analysis : Market-based IV for realistic calculations
  • Strike Prices : Real strike prices available in the market
  • Expiration Dates : Actual expiration dates for accurate time calculations
  • Market Prices : Real bid/ask prices for comparison with model prices

Disadvantages and Limitations

🔴 Limitations of Models

  • Black-Scholes Assumptions : Constant volatility, no jumps, constant risk-free rate
  • Limited Monte Carlo : Basic simulation, no advanced stochastic models
  • Limited Real Data : Manual parameters still used, Tradier integration for validation only
  • Static Greeks : Point calculations, no time-dependent dynamics
  • Performance : Monte Carlo slow for large simulations

🟡 Solutions for Improvement

  • Advanced Models : Heston, SABR, Local Volatility Models
  • Real Option APIs : Enhanced Tradier integration, CBOE, OCC, Bloomberg for market data
  • Implied Volatility Analysis : Full integration with Tradier for real-time IV calculations
  • Dynamic Greeks : Calculations in real-time with market movements
  • GPU Optimization : Utilization of CUDA for fast Monte Carlo

Data Integrity Tests

Parameter Validation

  • Verification that all prices are strictly positive
  • Control of bounds for risk-free rate and volatility (0-1)
  • Validation of maturity positive
  • Control of confidence level (0.5-0.999)
  • Limitation of number of displayed paths (0-200)

Calculation Validation

  • Control of consistency between Black-Scholes and Monte Carlo
  • Verification of NaN values in Greeks
  • Validation of confidence intervals
  • Measurement of calculation time for optimization
  • Handling of edge cases (zero volatility, very short maturity)

Validation of Trajectories

  • Control of consistency of Monte Carlo paths
  • Validation of time grid
  • Verification of distribution of final prices
  • Limitation of number of steps (max 300)
  • Memory management for simulations

Implemented Pricing Models

Black-Scholes (Analytical)

  • Closed-form Solution : Exact solution for European options
  • Analytical Greeks : Exact partial derivatives
  • Performance : Instantaneous (a few milliseconds)
  • Precision : Exact result under model assumptions
  • Limitations : Restrictive assumptions (constant volatility, etc.)

Monte Carlo (Simulation)

  • Stochastic Simulation : Geometric Brownian Motion
  • Numerical Greeks : Calculation by bump-and-revalue with CRN
  • Confidence Interval : Statistical estimation of error
  • Flexibility : Adaptable to more complex models
  • Performance : Time proportional to number of simulations

Calculated Greeks

Delta (Δ)

Sensitivity of option price to underlying asset price

Gamma (Γ)

Sensitivity of Delta to underlying asset price (convexity)

Theta (Θ)

Sensitivity of option price to time (time decay)

Vega (ν)

Sensitivity of option price to volatility

Rho (ρ)

Sensitivity of option price to risk-free rate

Technical Architecture

Backend (Python/NumPy)

  • Intensive Use of NumPy : Utilization of NumPy for performance
  • Common Random Numbers (CRN) : Reducing variance for Greeks
  • Monte Carlo Optimization : Terminal distribution for pricing, paths for visualization
  • Memory Management : Limiting paths and steps to avoid overload
  • Robust Validation : Consistency checks and error handling

Frontend (Chart.js/JavaScript)

  • Interactive Form : Real-time validation with visual feedback
  • Progress Bar : Animation with chips and timing
  • Monte Carlo Chart : Visualization of multiple trajectories with Chart.js
  • Distribution of Prices : Histogram of values of final paths
  • Responsive Design : Mobile and tablet adaptation

Implied Volatility Analysis Surface Page

Overview

"Implied Volatility Analysis Surface" page offers an interactive analysis of implied volatility based on real-world option data, with advanced 3D visualization and comparison of volatility smiles.

Main Features

  • 3D Volatility Surface : Interactive visualization in three dimensions with Plotly
  • Real-world Option Data : Utilization of Finnhub API
  • Comparison of Smiles : Comparative analysis between different stocks
  • Configurable Parameters : Range around spot (10% to 50%), selection of provider
  • Detailed Metrics : Current spot, IV min/max/average, complete statistics
  • Advanced Debug Mode : Visualization of raw data and IV matrices
  • API Monitoring : Real-time status of data providers

Used APIs

Tradier API (Primary)

  • Main Endpoint : /api/tradier/options/<symbol>
  • Data Retrieved : Complete option chains, spot price, implied volatility, Greeks
  • Advantages : Professional data, high accuracy, comprehensive coverage
  • Features : Real-time data, multiple expirations, detailed option information

Finnhub API (Fallback)

  • Endpoint : /api/vol-surface-3d/<symbol>
  • Data Retrieved : Basic option chains, spot price, implied volatility
  • Advantages : Free data, wide coverage
  • Limitations : Rate limits, sometimes incomplete data

Disadvantages and Limitations

🔴 Current Limitations

  • Tradier Rate Limits : 120 calls/minute, requires API key management
  • Data Quality : Sometimes incomplete data, missing strikes for some symbols
  • Maturity Limits : Limited expirations available, no long-term historical data
  • Coverage : Primarily US stocks, limited international coverage
  • Historical Data : Limited historical option data for backtesting

🟡 Solutions for Improvement

  • Enhanced Tradier : Tradier Pro for higher rate limits and better data
  • Paid APIs : Bloomberg Terminal, Reuters Eikon, CBOE DataShop
  • Professional Data : IEX Cloud Pro, Alpha Vantage Pro
  • Local Database : Storing historical volatility surfaces
  • Calibration Models : Interpolation and extrapolation of missing data
  • Alternative APIs : Quandl, FRED for macroeconomic data
  • Machine Learning : Volatility prediction with ML models

Technical Glossary

API Terms

Endpoint

Specific access point to a web API allowing data exchange. Example: /api/market-data to retrieve market data.

User-Agent

HTTP header identifying the client software making the request. Used to simulate a web browser and avoid API blocking.

Rate Limiting

Limitation of the number of allowed requests per unit of time (e.g., 60 calls/minute) to prevent server overload.

JSON

Lightweight and readable data format used for exchanging information between web applications. Hierarchical structure with keys and values.

HTTP Status Code

Numeric code indicating the result of an HTTP request: 200 (success), 404 (not found), 429 (too many requests), 500 (server error).

Financial Terms

Implied Volatility Analysis

Measure of market uncertainty derived from option prices. Higher = more uncertainty about the price of the underlying asset.

Volatility Surface

3D representation of implied volatility based on strike price (exercise price) and maturity (time to expiration) of options.

Greeks

Measures of sensitivity of an option's price: Delta (price), Gamma (delta), Theta (time), Vega (volatility), Rho (interest rate).

Call/Put

Types of options: Call (right to buy), Put (right to sell) an asset at a fixed price before a certain date.

Strike Price

Fixed exercise price in the option contract at which the asset can be bought or sold.

VaR (Value at Risk)

Measure of risk indicating the maximum potential loss on a portfolio with a given level of confidence (e.g., 95%).

Max Drawdown

Maximum loss recorded from the peak historical investment, measuring risk of decline.

Sharpe Ratio

Measure of risk-adjusted performance: excess return divided by portfolio volatility.

Technical Terms

Backend

Server part of the application handling data, performing calculations, and managing business logic (Python/Flask).

Frontend

Client-side user interface displaying data and allowing interaction (HTML/CSS/JavaScript).

API (Application Programming Interface)

Set of rules allowing different software to communicate and exchange data in a standardized manner.

RESTful

Architecture style for web APIs using standard HTTP methods (GET, POST, PUT, DELETE) to manipulate resources.

Monte Carlo

Numerical simulation method using random numbers to solve complex mathematical problems.

ThreadPoolExecutor

Pool of threads allowing multiple tasks to run in parallel to improve API request performance.

Fallback

Automatic standby mechanism activated automatically in case of failure of a main service (e.g., switching to an alternative API).

Timeout

Maximum waiting time for an operation (e.g., 5 seconds for an API request) before considering it failed.

Data Terms

Timeframe

Period of time for data aggregation: 1 day, 1 week, 1 month, etc. Determines the granularity of the graphs.

OHLC

Price data: Open (opening), High (highest), Low (lowest), Close (closing) for a given period.

Volume

Number of trades during a period, indicator of market activity and liquidity.

Return

Relative change in price of an asset over a period, expressed as a percentage. Measure of performance.

Normalization Base 100

Method for comparing instruments with different prices by bringing them to a common base of 100.

Correlation

Measure of linear relationship between two variables, ranging from -1 (perfect negative correlation) to +1 (perfect positive correlation).

Visualization Terms

Chart.js

JavaScript library for creating interactive and responsive graphs in web applications.

Plotly

Scientific visualization library allowing creation of interactive 3D graphs and advanced dashboards.

Logarithmic Scale

Type of scale where distances represent ratios rather than differences, useful for visualizing relative changes.

Tooltip

Information window that appears on hover over a graphical element, displaying additional details.

Responsive Design

Web design approach allowing the interface to automatically adapt to different screen sizes.

Wireframe

3D display mode showing only the surface structure without filling, useful for geometric analysis.
